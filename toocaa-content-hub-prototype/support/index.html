Why a unified Support entry matters

The current site already has multiple “support-like” destinations (Help Center pages, Wiki docs, Blog tutorials). The missing piece is a single routing layer that makes the experience predictable for users — and crawlable for search engines.

User flow
  • Before purchase: learn, compare, validate safety & requirements.
  • After purchase: install, assemble, first project, maintenance.
  • When something breaks: troubleshooting tree + ticket template.
Content governance
  • Use SSOT: one “main” URL per topic, then 301/canonical for duplicates.
  • Each page should contain: TL;DR, steps, tables, FAQ, CTA.
  • Connect to products and accessories where relevant.
