Array tests: the fastest way to build a settings library

A simple process: define a matrix, run it, choose best cells, and publish a settings card to the canonical material page.

Download Array Test CSVGo to Materials & Settings

Why array tests are the scalable growth engine

Materials & Settings pages become “programmatic SEO” when you can standardize how you test and document new materials. The output is not just a blog post — it’s a reusable settings card (table + notes).

  • Fast learning loop for new materials or new batches.
  • Creates structured data that AI systems can quote.
  • Reduces support load by turning “it burns” into a testable matrix.

Workflow

StepActionDeliverable
1Pick the material + thickness + goal (engrave/cut).Test plan
2Create a small grid (power × speed × passes).Matrix definition
3Run test with stable focus + airflow conditions.Raw results
4Pick best cells, write notes (edge quality, residue, melt).Settings card
5Publish to the material page (SSOT) and link from use cases.Canonical page update

Where it lives in the IA

  • Materials & Settings: the canonical home for settings cards.
  • Use Cases: link to the settings card for each featured material.
  • Troubleshooting: link to the “quality issues” section of the material page.
