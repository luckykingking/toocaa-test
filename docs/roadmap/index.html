1-year plan (high level)

The purpose of this roadmap is to make content delivery predictable. The first 90 days focus on fixing structure and adding 3–5 canonical pages. The following months scale with templates (use cases, materials, troubleshooting).

0–90 days (foundation)

  • Confirm SSOT rules and publish the Link Pool (canonical / 301 / noindex decisions).
  • Build 3 pillar pages: Before You Buy, After You Buy, Troubleshooting hub.
  • Publish 1 full Use Case page (e.g., Pro Studio / Crafts) + 2 Tutorials.
  • Define the Materials & Settings page template + Array Test workflow.

4–12 months (scale)

  • Publish 2–4 use cases per month (beginner, classroom, maker, small business, gifts, signage).
  • Turn top support tickets into troubleshooting trees (quality, alignment, software).
  • Grow Materials & Settings library with programmatic templates (validated by array tests).
  • Measure SEO + Support + GEO: organic clicks, self-solve rate, AI citations.
