SSOT: Single Source of Truth for URLs

When the site has multiple destinations (Shopify pages + Blog + Wiki), the fastest way to lose SEO/GEO is duplicated topics across URLs. This page is a governance tool: define one canonical URL for each topic, and decide 301/canonical/noindex for others.

Rules
  • Canonical for the main page.
  • 301 redirect for true duplicates.
  • Noindex for thin pages that should exist for UX only.
  • Keep a maintained “Link Pool” sheet as SSOT.
Operational habit
  • New blog post? Link it to a pillar/use-case/material page.
  • New wiki doc? Add it to the Link Pool and define canonical.
  • Every month: prune duplicates and update internal links.
