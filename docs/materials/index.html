Materials & Settings: recommended page structure (SSOT)

Each material page should be a reusable reference (settings table + explanations + quality troubleshooting + safety + related links). This is the most scalable content type for both SEO and GEO.

Per-material page blocks
  • Recommended settings table (copyable)
  • Variable glossary (power, speed, passes, focus, air assist, masking)
  • Quality troubleshooting (burn, melt, haze, burrs)
  • Safety notes (fumes, ventilation)
  • Related tutorials + relevant accessories + CTAs
Scalable publishing workflow
  • Maintain settings as a CSV / sheet (single source).
  • Generate pages from structured data (optional).
  • Use Array Test templates to validate new materials.
  • Link every use case and troubleshooting page back to the canonical material page.
